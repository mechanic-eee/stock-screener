## 0. 이 시스템은 무엇이고, 무엇이 아닌가

**무엇이다**: 5년 고가 대비 -50% 이상 폭락한 KR+US 종목을 매일 스캔해, 검증된 펀더멘털
신호(부도회피·퀄리티·저변동성)로 랭킹하고, 주간 깔때기로 "사람이 확인할 후보"까지 좁혀주는
**의사결정 보조 시스템**. 엣지는 백테스트로 검증됨(rank IC t4~7, KR 생존편향 보정).

**무엇이 아니다** (`docs/service-audit-2026-07-19.md` §5 — 절대 주장 금지 10항 요약):
- ❌ "점수 높으면 사라"는 시스템이 아니다 — 검증된 건 **시장 내 상대 랭크**뿐, 절대 점수는 표류한다.
- ❌ 절대수익 약속이 아니다 — 엣지 +1.3~3.8%p/픽은 **같은 날 폭락주 평균 대비**이고, 절대수익은
  레짐(200일선)이 지배한다(위 250d −2.8% vs 아래 −16.6%).
- ❌ "게이트 통과 = 안전"이 아니다 — 통과는 "탈락 사유를 못 찾음"이다.
- ❌ 손절이 파국을 막아주지 않는다 — 갭·거래정지·상폐(−100%)의 진짜 상한은 **최대비중 15%**다.
- ❌ "알림 없음 = 이상 없음"이 아니다 — 그래서 매일 하트비트가 온다(§2).

**현실적 기대치**: 픽당 시장(폭락주 코호트) 대비 +1.3~3.8%p, 보유 120~250일, 승률 ~60%,
비용(왕복 KR ~0.45%/US ~0.3%+양도세 22%)이 엣지의 일부를 먹는다. 1~2년 마이너스 구간도
정상 범위 — 그래서 슬리브는 계좌의 60% 이하.

## 2. 요일별 사용법

### 화~토 아침 — 스캔 알림 읽기 (30초)
읽는 순서: ①**⚠️ 신선도 경고 있으면 그날 데이터 불신** ②**건강라인**(시세일·펀더%·신규신호%) —
⚠️ 붙으면 품질 저하 ③**시장: 레짐** — `200일선↓(주의)` 뜬 시장은 신규 진입 금지(지금 KOSPI
폭락 중이라 KR 이탈 가능성 주시) ④상위 15는 **참고만** — 매수 후보는 주말 깔때기가 정한다.
핵심% 태그(펀더가 받치는 점수인가)가 낮은 고점수는 의심.

### 월~금 08:10 이후 — 감시 결과 (폰 확인)
- **알림 없음 + 🩺 하트비트 옴** = 정상 (보유 n·최소 손절여유·다음 이벤트 요약)
- **알림 옴** = §3 대응
- **아무것도 안 옴** = 감시가 죽었다 → §6 고장 대응

### 월요일 — 📊 주간 성적표 (1분)
코호트 vs KOSPI/S&P 비교가 핵심: "우리가 벌었나"가 아니라 **"시장 대비 어디인가"**를 본다
(예: 6/25 코호트 +2.9% vs KOSPI −23.6% = 대폭 초과성과). 단서 문구대로 비용·세금 미반영 참고치.

### 주말 — 사이클 (핵심 의식, ~1.5h)
```powershell
cd C:\Users\yoobg\Claude_work\projects\stock-screener
.venv\Scripts\python.exe scripts\recommend.py          # 깔때기 (~5분, 위험공시 재점검 포함)
```
1. 헤더 확인: **시세 기준일**(낡으면 스스로 중단함) · 계좌 가정 · 레짐 · 비용.
2. `exports\recommend-날짜.md` 열고 **⭐ 우선 리서치(시장별 5)부터** 체크리스트 — 종목당 15분,
   딥링크(DART/EDGAR/뉴스) 클릭, ⚠️ 섹터 중복·유동성 미검증·위험공시 미수행 줄 확인.
   목적은 **탈락 사유 찾기**(negative screen). 베토는 주 2건 상한.
3. 판정 기록 (전부 기록해야 베토 실효성 검증이 산다):
```powershell
.venv\Scripts\python.exe scripts\decide.py --ticker XXX --action 관망 --note "[베토] 사유"   # 탈락
.venv\Scripts\python.exe scripts\to_watchlist.py --tickers XXX,YYY                          # 채택 시드
# WATCHLIST에서 논거·촉매 채우고:
.venv\Scripts\python.exe scripts\decide.py --ticker XXX --paper                             # 페이퍼 기록
```
4. 픽 규칙: 3~5종목 · 시장 최대 3 · 섹터당 1 · **확정 즉시 진입**(눌림 대기 = 검증된 손해) ·
   2차 트랜치 +2~4주 시간 기반 · 기존 코호트 소속 종목(현재 NVO·AMN)은 픽 제외(실험 무결성).
5. 캘린더에 2차 트랜치·120d 리뷰 등록 (monitor D-3 알림이 백업).

## 3. 알림 기호 → 대응 행동

| 기호 | 뜻 | 당신의 행동 |
|---|---|---|
| 🔴 손절이탈 | 현재가 ≤ 손절선 | **당일 청산이 기본** — `decide.py --ticker X --action 청산 --exit 가격 --note "손절"` |
| 🔴 가격조회 실패 | 거래정지·상폐 가능성 | 즉시 종목 조회 — 이 유니버스 최악 시나리오의 전조 |
| 🔴 위험공시/감사의견/자본잠식 | KR 치명 공시 | 당일 청산 검토 (thesis-break) |
| 🔴 EDGAR 신규 424B/S-1/25/NT | US 희석·상폐·지연 | 공시 원문 확인 → thesis-break면 청산 |
| 🟠 EDGAR 신규 8-K | US 중요 이벤트 | 내용 확인 (실적·계약 등 — 중립일 수 있음) |
| 🟠 랭킹 하위 50% 강등 | 엣지 근거 소멸 | thesis 재평가 — SMCI 패턴 |
| 🟠 손실 −30% | 깊은 손실 | 논거 재점검 |
| ℹ️ 상위 25% 밖 | 2차 트랜치 취소 조건 | 2차 집행 안 함 (1차는 유지) |
| ℹ️ 후보 이탈 | 낙폭 <50% 회복 가능성 | 🎓 졸업인지 확인 — 좋은 신호일 수 있음 |
| ⚪ 미수행 | 그 검사가 **안 된 것** (통과 아님!) | 수동 확인 또는 다음 실행 대기 |
| 🗓 D-n | 이벤트 임박 | 해당 결정 준비 (2차 트랜치·실적) |

## 4. 명령어 치트시트

```powershell
# 전부 projects\stock-screener 에서, .venv\Scripts\python.exe 로 실행
scripts\recommend.py                     # 주간 깔때기 (--no-fresh-distress 빠름, --allow-stale 강행)
scripts\to_watchlist.py --tickers A,B    # 워치리스트 시드 (ATR 손절 자동초안)
scripts\decide.py --ticker X --paper     # 페이퍼 매수 기록 (--dry-run 미리보기)
scripts\decide.py --ticker X --action 관망 --note "..."       # 베토/보류 기록
scripts\decide.py --ticker X --action 청산 --exit 12.34 --note "..."
scripts\position_size.py --entry 100 --stop 90                # 사이징 단독 계산
scripts\track.py                         # 추적 갱신 (TRACKING.md, 벤치마크 병기)
scripts\monitor.py                       # 감시 1회 (--weekly 성적표 강제, --telegram 전송)
scripts\daily.ps1                        # track+monitor 한 번에 (-Telegram)
scripts\selftest.py                      # 검증된 설정 회귀 확인
scripts\register-daily-task.ps1          # 08:10 예약작업 재등록 (-Unregister 해제)
```

## 5. 앱 사용법 (수시 탐색)

- 사이드바 지표는 **🟢 핵심 그룹 마스터 체크 1클릭** = 검증된 레시피. ⚪ 그룹은 예측력 없음
  (백테스트 확인) — 켜는 건 자유지만 랭킹 신뢰는 핵심 그룹 기준.
- 행 클릭 → **점수 분해**: 펀더가 받치는 점수인지, 데이터 없음 지표가 몇 개인지 먼저 본다.
- 헤더 레짐 배지·신선도 배너 경고 시 그날 판단 보류.
- 결과표는 탐색용 — 실행 파이프라인은 반드시 recommend.py 경유(위험공시 재점검이 붙는다).

## 7. 페이퍼 → 실전 전환 로드맵

1. **~9/12 (8주)**: 페이퍼로 전 파이프라인 집행. 판단 기준은 수익률이 아니라 **규칙 준수율**
   (12주 40픽으로 엣지 판독은 통계적으로 불가 — 감사 확정).
2. **실돈 시작 시**: `data\portfolio.json` 생성(gitignored) —
   `{"account_krw": 실계좌, "account_usd": 실계좌, "risk_pct": 0.5, "max_pos_pct": 15}` —
   **첫 구간 R 0.5%(반액)**, 체결·슬리피지 마찰 확인 후 1.0%로.
3. **매수 기록은 `--paper` 빼고 동일 명령** — 페이퍼/실계좌는 track이 자동 분리 집계.
4. **10/10 분기 리뷰**: 12주 데이터로 규칙 개정(랭크컷·ATR 문턱 등 이연 안건 포함).
   그 전에 규칙 만지면 forward 검증 오염.
