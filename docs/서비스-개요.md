# 폭락주 스크리너 — 서비스 개요 (한눈에 보고 활용)

> **무엇:** 5년 고가 대비 폭락한 종목(KR+US) 중, **회복 가능성 높은 쪽을 점수로 줄 세워** 발굴→결정→사이징→추적까지 잇는 투자 보조 서비스.
> **핵심:** 점수는 *매수 신호가 아니라 랭킹 필터*다. 그리고 그 랭킹은 **백테스트로 실제 검증됐다**(아래).

---

## 1. 30초 요약 — 검증으로 확인된 진실

| 질문 | 검증된 답 |
|---|---|
| 폭락주를 그냥 사면? | **동전던지기**(120d 평균≈0%, 승률~40%). 모집단 자체엔 엣지 없음. |
| 그럼 점수가 하는 일은? | 그 안에서 **회복주 vs falling knife를 줄 세움**. 상위10이 코호트 평균 대비 **+1~4%p/픽**(KR, 대부분 분기 양+). |
| 무엇이 엣지를 만드나? | **차분함(저ATR) + 부도회피(Altman) + 퀄리티(GP/Piotroski/재무)**. ← 이게 "enrichment". |
| 펀더가 진짜 도움되나? | **그렇다(직접 검증).** 펀더 넣으면 합성 IC가 거의 2배(KR t1.9→t5.7). 가격만 점수는 최상위서 역전, 펀더가 그걸 고침. |
| 언제 사야 하나? | **시장이 200일선 위(상승국면)일 때.** KR: 200일선 위 −2.8% vs 아래 −16.6%(250d). |
| 얼마나 믿나? | 통계로 이긴다(개별 노이즈 큼, 최악 −100%). **분산·손절·시간분산 필수.** |
| 안 통한 것은? | **accruals**(이익의질) — 폭락주선 역설(손상기업 신호)이라 점수서 뺐다. |

상세 근거: [점수 활용 가이드](점수-활용-가이드.md) · [검증 종합](score-validation-2026-06-27.md) · [전략 백테스트](strategy-backtest-top10.md) · [레짐 분석](regime-analysis.md).

## 2. 어떻게 쓰나 — 일일 루프

```
 ① 발굴            ② 선별            ③ 결정+사이징         ④ 추적/리뷰
 앱/텔레그램 알림 → 워치리스트     → decide.py        →   track.py
 (점수 상위)        (관심종목 큐레)   (수량·DECISIONS)      (수익률·점수실효성)
```
- 매일: 텔레그램 알림 확인(점수 상위 + **건강라인**[데이터 신선도] + **시장 레짐**[200일선]).
- 살 때: `python scripts/decide.py --ticker <T>` → 워치리스트 행 파싱→ATR손절 사이징→DECISIONS 기록 한 번에.
- 사후: `python scripts/track.py` → 점수가 실제 수익을 예측했나 추적.
- 전체 운영 매뉴얼: [일일 워크플로](../../stock-investing/워크플로.md).

## 3. 점수는 어떻게 만들어지나
- **기본 게이트:** 5년 고가 대비 −50%↓ + 유동성 하한(잡주·페니 제거).
- **합성점수 = Σ(가중치 × 신호점수) / Σ가중치** (보조지표 토글 시). 신호별 가중치는 **백테스트 IC로 보정됨**:
  - 🟢 강: `atr_risk`(0.20, 최강), `fundamental`(0.25), `altman_z`(0.18), `piotroski`(0.20), `gross_profit`(0.10)
  - 🟡 약: `relative_strength`·`weekly_macd`(0.05), `rsi`·`bollinger`·`moving_average`(0.05)
  - ⚪ 제외(가중0): `obv`·`volume_surge`·`accruals` (예측력 0/음수로 검증됨)
- 앱 "🔍 점수 분해"에서 종목별 기여를 본다. **랭킹엔 반드시 펀더 포함**(가격만은 최상위 역전).

## 4. 현실적 기대치 (과대평가 금지)
- 엣지는 **modest** — 픽당 +1~4%p(KR). 큰돈은 *여러 종목·여러 번·몇 달 홀드*의 누적에서.
- **절대수익은 타이밍 의존** — 나쁜 시장창(지수 하락)에선 최고점수도 무더기로 깨진다. → 200일선 레짐 + 시간 분산.
- **개별 보장 없음** — 최악 −100%(상폐). 한두 종목 몰빵 금지.
- **US 수치는 참고만**(표본 얇음·상폐 누락). 신뢰의 무게중심은 **KR**(상폐보정·28날짜).

## 6. 검증의 신뢰성 — 왜 믿나
- 백테스트가 **production 필터를 그대로 호출**(과거 시점 데이터에) → "백테스트 점수 = 라이브 점수". 재구현 착시 없음.
- 펀더는 **point-in-time**(US=SEC EDGAR `filed≤T` / KR=DART 사업보고서) → look-ahead 없음.
- **적대적 멀티에이전트 검증**이 한 차례 EDGAR 데이터버그를 잡아내 결론을 정정함(검증된 검증). 자세히: [검증 종합](score-validation-2026-06-27.md) "적대적 검증" 절.
