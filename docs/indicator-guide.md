# 보조지표 사용 가이드

> 스크리너의 보조지표를 **언제·어떻게 쓰는지** 정리한 실전 문서.
> 지표 정의는 코드(`src/screener/filters/*.py`) 기준 (2026-06-23 갱신 — 부도/퀄리티/이익질 지표 추가).
> 앱 메인의 "ℹ️ 점수는 어떻게 매겨지나" 패널과 같은 내용.

---

## 0. 작동 원리 (핵심)

기본 스크린(**고가 대비 폭락**, 5년 고가 대비 −50%)이 항상 켜져 있고, 그 통과 종목 위에 보조지표를 얹는다. 모든 지표는 **두 역할을 동시에** 한다:

1. **스코어러(점수 기여):** 0~100점을 매겨 **가중평균**으로 합성점수(`점수` 칼럼)를 만든다. 켜진 지표가 많을수록 순위가 정교해진다. (안 켜면 기본 폭락 점수뿐 → 동점 다수)
2. **게이트(제외):** 임계를 넘기면 종목을 **탈락**시킨다.

**`통과 최소 점수(min_score)` = 스코어러 ↔ 게이트 전환 스위치**
- `min_score = 0` (기본): 제외 없이 **점수만** 기여 → 순위에만 영향
- `min_score`를 올리면: 그 미만 종목을 **제외** → 후보 수가 줄어든다

> **기본 하락률 슬라이더도 게이트**다(2026-06-23 수정). 스냅샷은 −50%로 사전 스크리닝돼 있어 50% 아래로 내려도 더 늘지 않고, 위로 올리면 좁혀진다.

앱에서 지표를 켜면 expander에 **파라미터 + 가중치 슬라이더**가 자동으로 뜬다. 가중치를 올리면 그 지표가 순위에 더 크게 반영된다. 사이드바 지표는 **선택 우선순위 4그룹**(🟢핵심 → 🔵보강 → 🟡확증·타이밍 → ⚪예측력 없음)으로 묶여 구분선으로 나뉘어 있다 — 위 그룹부터 켜면 된다. 그룹마다 **전체 켜기/끄기** 마스터가 있어 🟢 핵심은 한 클릭.

결과 표의 **`핵심기여` 칼럼**은 점수 중 🟢 엣지 신호의 비중(%) — "펀더가 받치는 점수인가"를 표에서 바로 가른다. **행을 클릭**하면 아래 `🔍 점수 분해` 패널에 요소별 기여 차트(티어 색), 지표별 점수·상세, **ATR 손절 초안가**, 데이터 없는 지표 목록까지 나온다. 헤더의 **200일선 레짐 배지**(KR/US)가 '언제'를, 점수가 '무엇'을 답한다.

> **신뢰 주의:** RS·밸류·펀더(F-score·Altman 등)·뉴스는 외부 데이터라, 데이터를 못 받으면 **중립 50점**으로 처리되고(순위 영향 없음) 사이드바·상단에 경고가 뜬다. KR/US 밸류·펀더는 스냅샷 prime으로 작동(신규 F-score·Altman은 다음 daily-scan부터 채워짐), 뉴스는 KR(네이버)만 작동(US는 NEWSAPI_KEY 필요).

## 1. 지표 카탈로그 (사이드바 그룹 = 선택 우선순위)

그룹과 가중치는 **2026-06 점수 검증**(`score-validation-2026-06-27.md`, per-date IC로 보정)을 그대로 반영한다. 설계 원칙은 "서로 겹치지 않는 차원을 채운다".

### 🟢 핵심 — 항상 켜기 (검증된 엣지, 두 시장 t4~7)
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **펀더멘털** | 가치함정 자동제외 — **치명(단독제외)**: 자본잠식·4Q적자(US)·**감사의견 비적정**·부도/회생 등 DART 위험공시(KR) / **약신호(누적)**: 매출급감·고부채. + 재무 점수. *KR 최강(t5~7)* | **0.25** |
| **피오트로스키 F** | 9점 회계건전성 — 재무가 *개선 중인가*(추세). 폭락 저평가주 중 돌아설 놈 선별. *양 시장 t2.3~3.6* | 0.20 |
| **ATR 리스크/손절** | 일봉 변동성(ATR%) 낮을수록↑ + 권장 손절폭(2.5×ATR). **최강 가격신호**(IC t3~5.5) — 저변동 폭락주가 살아남는다 | 0.20 |
| **알트만 Z(부도위험)** | Z'' 부도거리. 폭락주는 부도위험이 높아 *추가 배제*가 핵심. 기본은 점수만(자동제외 OFF). *t2.7~5.1* | 0.18 |
| **퀄리티(매출총이익률)** | 매출총이익/총자산(연율화) = 구조적으로 돈 버는 사업인가. *t4.4~6.7* | 0.10 |

### 🔵 보강 — 함께 켜면 좋음 (게이트·희석·바닥구조)
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **밸류에이션** | 싸고(낮은 PER/PBR) 우량한가(ROE/배당). min_score 게이트로 "안 싼 것" 제외용 | 0.15 |
| **발행주식수(희석/자사주)** | 주식수 변화 — 자사주매입(호재) vs 증자·CB 희석(악재). *US 검증 t5.1, 현재 US만* | 0.10 |
| **변동성 수축(VCP)** | 밴드폭이 과거 분포 대비 좁아짐 = **바닥 다지는 중** (falling knife 회피) | 0.10 |

### 🟡 확증·타이밍 — 약신호 (검증서 IC≈0~시장의존, 낮은 가중)
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **상대강도(RS)** | 종목 수익률 − 시장지수(US=S&P500/KR=KOSPI). *IC≈0으로 검증돼 강등* | 0.05 |
| **주봉 MACD** | 일봉→주봉 리샘플, 상위 시간프레임 추세 확인. *IC≈0으로 검증돼 강등* | 0.05 |
| **MACD 음→양 전환** | 최근 N일 내 0선/시그널선 상향돌파, 최근일수록↑ | 0.10 |
| **RSI 임계** | 과매도(기본 35↓). *폭락주는 거의 항상 켜져 변별력 낮음, 시장의존* | 0.05 |
| **볼린저밴드 위치** | %B로 밴드 하단 부근. *RSI와 중복, 시장의존* | 0.05 |
| **이동평균선** | 종가가 N일선 위/상향돌파. *60d 이후 IC 소멸* | 0.05 |
| **긍정 뉴스 증가** | 최근 기사 수 + 평균 감성. *감성원이 placeholder라 저가중* (KR=네이버·US=NewsAPI) | 0.10 |
| **카탈리스트(실적)** | 실적 임박(7일내) ⚠️경고 + (선택)실적 직후 MACD전환 보너스. *보너스 기본 OFF* | 보너스 |

### ⚪ 예측력 없음 — 꺼두기 권장 (검증 음성, 가중 0)
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **OBV 누적매집** | 지속적 매집. *양 시장 음의 IC(KR t−2.3~−3.0) — anti-predictive* | 0 |
| **거래량 급증** | 단기/장기 평균 배수. *폭락주의 거래량 급증은 투매 신호로 읽힘 — 약음* | 0 |
| **이익의 질(발생액)** | (순이익−영업현금흐름)/자산. *두 클린표본서 무력/해로움(KR t−3.5) — 컷* | 0 |

## 2. 실전 조합 (레시피)

**A. "검증된 엣지 그대로" (추천 출발점 = 🟢 핵심 5종)**
- 켜기: `펀더멘털` + `피오트로스키 F` + `ATR 리스크/손절` + `알트만 Z` + `퀄리티(매출총이익률)`
- 의도: 백테스트로 확증된 조합 그대로 — 부도회피(Altman)·개선추세(F)·저변동(ATR)·수익구조(GP)·가치함정 제거(펀더). 일일 알림 랭킹과 같은 설정. 가중치는 기본값(IC 보정치) 유지 권장.

**B. "싸구려 함정 빼고 진짜 저평가 우량주" (A + 🔵 보강)**
- 켜기: A 조합 + `밸류에이션`(min_score 60↑로 비싼 것 게이트) + `발행주식수`(US 희석 회피) + `변동성 수축(VCP)`
- 의도: 핵심 엣지 위에 가치함정 게이트·희석 페널티·바닥구조 확인을 얹는다.

**C. "후보를 강하게 압축"**
- 한두 지표의 `통과 최소 점수`를 60~70으로 올려 게이트화 → 후보 수 급감.

**가중치 운용:** 기본값이 이미 IC로 보정된 값이라 **그대로 쓰는 게 검증된 설정**이다. 🟡 확증 지표는 켜도 낮은 가중(0.05)만 두고, ⚪ 그룹은 켜지 않는다. **가격 지표만으로 랭킹하지 마라** — 검증에서 최상위 분위가 역전됐다(펀더 필수, `점수-활용-가이드.md` 참고).

## 3. 두 가지 경로

- **앱(인터랙티브):** 사이드바 토글 → 즉시 재계산. 탐색·튜닝에 최적. 바탕화면 "폭락주 스크리너" 아이콘으로 실행(최신 클라우드 스냅샷 로드).
- **워치리스트 시드(`scripts/to_watchlist.py`):** 같은 지표 엔진으로 상위 N을 골라 워치리스트로 전송.
  - 기본 랭킹 = **일일 알림과 같은 검증 세트**(`fundamental valuation altman_z piotroski gross_profit atr_risk`) — 그냥 `python scripts/to_watchlist.py --top 10`이면 된다. base-only는 `--base-only`(비권장, 동전던지기).
  - `--list-indicators`로 지표 키 확인, `--weights key=w,...`로 가중 오버라이드, `--dry-run`으로 미리보기.
