# 보조지표 사용 가이드

> 스크리너의 보조지표를 **언제·어떻게 쓰는지** 정리한 실전 문서.
> 지표 정의는 코드(`src/screener/filters/*.py`) 기준 (2026-06-23 갱신 — 부도/퀄리티/이익질 지표 추가).
> 앱 메인의 "ℹ️ 점수는 어떻게 매겨지나" 패널과 같은 내용.

---

## 0. 작동 원리 (핵심)

기본 스크린(**고가 대비 폭락**, 5년 고가 대비 −50%)이 항상 켜져 있고, 그 통과 종목 위에 보조지표를 얹는다. 모든 지표는 **두 역할을 동시에** 한다:

1. **스코어러(점수 기여):** 0~100점을 매겨 **가중평균**으로 합성점수(`점수` 칼럼)를 만든다. 켜진 지표가 많을수록 순위가 정교해진다. (안 켜면 기본 폭락 점수뿐 → 동점 다수)
2. **게이트(제외):** 임계를 넘기면 종목을 **탈락**시킨다.

**`통과 최소 점수(min_score)` = 스코어러 ↔ 게이트 전환 스위치**
- `min_score = 0` (기본): 제외 없이 **점수만** 기여 → 순위에만 영향
- `min_score`를 올리면: 그 미만 종목을 **제외** → 후보 수가 줄어든다

> **기본 하락률 슬라이더도 게이트**다(2026-06-23 수정). 스냅샷은 −50%로 사전 스크리닝돼 있어 50% 아래로 내려도 더 늘지 않고, 위로 올리면 좁혀진다.

앱에서 지표를 켜면 expander에 **파라미터 + 가중치 슬라이더**가 자동으로 뜬다. 가중치를 올리면 그 지표가 순위에 더 크게 반영된다. 사이드바 지표는 **위에서부터 중요한 순서**로 정렬돼 있다.

> **신뢰 주의:** RS·밸류·펀더(F-score·Altman 등)·뉴스는 외부 데이터라, 데이터를 못 받으면 **중립 50점**으로 처리되고(순위 영향 없음) 사이드바·상단에 경고가 뜬다. KR/US 밸류·펀더는 스냅샷 prime으로 작동(신규 F-score·Altman은 다음 daily-scan부터 채워짐), 뉴스는 KR(네이버)만 작동(US는 NEWSAPI_KEY 필요).

## 1. 지표 카탈로그 (중요도 = 사이드바 순서)

설계 원칙이 **"서로 겹치지 않는 차원을 채운다"**라, 차원별로 보면 이해가 쉽다.

### 🥇 가치함정 거르기 — 폭락엔 이유가 있다 (외부 재무 데이터)
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **펀더멘털** | 가치함정 자동제외 — **치명(단독제외)**: 자본잠식·4Q적자·**감사의견 비적정**·부도/회생 등 DART 위험공시(KR) / **약신호(누적)**: 매출급감·고부채. + 재무 점수 | **0.25** |
| **알트만 Z(부도위험)** | Z'' 부도거리. 폭락주는 부도위험이 높아 *추가 배제*가 핵심. 기본은 점수만(자동제외 OFF) | 0.18 |
| **피오트로스키 F** | 9점 회계건전성 — 재무가 *개선 중인가*(추세). 폭락 저평가주 중 돌아설 놈 선별 | 0.20 |

### 🥈 싸고 좋은가 — "빠진 것 ≠ 싼 것"
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **밸류에이션** | 싸고(낮은 PER/PBR) 우량한가(ROE/배당) | 0.15 |
| **퀄리티(매출총이익률)** | 매출총이익/총자산 = 구조적으로 돈 버는 사업인가(죽은 사업 회피) | 0.10 |
| **이익의 질(발생액)** | (순이익−영업현금흐름)/자산. 흑자전환이 진짜 현금인가 회계착시인가 | 0.10 |
| **발행주식수(희석/자사주)** | 주식수 변화 — 자사주매입(호재) vs 증자·CB 희석(악재). *현재 US만* | 0.10 |

### 🥉 바닥 다지고 도는가 — falling knife 회피 + 전환 타이밍
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **상대강도(RS)** | 종목 수익률 − 시장지수(US=S&P500/KR=KOSPI). 시장보다 덜 빠지거나 먼저 돌면↑ | 0.15 |
| **변동성 수축(VCP)** | 밴드폭이 과거 분포 대비 좁아짐 = **바닥 다지는 중** | 0.10 |
| **주봉 MACD** | 일봉→주봉 리샘플, **상위 시간프레임** 추세 확인 | 0.15 |
| **MACD 음→양 전환** | 최근 N일 내 0선/시그널선 상향돌파, 최근일수록↑ | 0.10 |

### 📊 수급/매집 · 🛡 리스크
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **거래량 급증** | 단기/장기 평균 배수. 3~5x 최고점, 10x↑ 이상치 플래그 | 0.10 |
| **OBV 누적매집** | 스파이크가 아닌 **지속적** 매집 | 0.10 |
| **ATR 리스크/손절** | 일봉 변동성(ATR%) + 권장 손절폭(2.5×ATR). 사이징·falling knife 경고 | 0(정보성) |

### 🔁 확증 / 약신호 (다른 지표와 차원 겹침 — 보조용)
| 지표 | 무엇을 보나 | 가중 |
|---|---|---|
| **RSI 임계** | 과매도(기본 35↓). *폭락주는 거의 항상 켜져 변별력 낮음* | 0.10 |
| **볼린저밴드 위치** | %B로 밴드 하단 부근. *RSI와 중복* | 0.10 |
| **이동평균선** | 종가가 N일선 위/상향돌파. *MACD와 중복* | 0.10 |
| **긍정 뉴스 증가** | 최근 기사 수 + 평균 감성. *감성원이 placeholder라 저가중* (KR=네이버·US=NewsAPI) | 0.10 |
| **카탈리스트(실적)** | 실적 임박(7일내) ⚠️경고 + (선택)실적 직후 MACD전환 보너스. *보너스 기본 OFF* | 보너스 |

## 2. 실전 조합 (레시피)

**A. "낙폭 큰데 진짜 돌아서는 우량" (추천 출발점)**
- 켜기: `펀더멘털` + `알트만 Z` + `피오트로스키 F` + `상대강도(RS)` + `변동성 수축(VCP)`
- 의도: 펀더/Altman으로 망해가는 회사 배제, F-score로 개선 추세, RS/VCP로 바닥·전환. 가중치는 취향대로.

**B. "싸구려 함정 빼고 진짜 저평가 우량주"**
- 켜기: `펀더멘털`(자동제외) + `밸류에이션`(min_score 60↑로 비싼 것 게이트) + `퀄리티(매출총이익률)`
- 의도: 펀더가 가치함정 제거, 밸류 게이트가 안 싼 종목 제외, GP가 죽은 사업 회피.

**C. "후보를 강하게 압축"**
- 한두 지표의 `통과 최소 점수`를 60~70으로 올려 게이트화 → 후보 수 급감.

**가중치 운용:** 중시하는 차원의 가중치를 올린다. 펀더멘털(0.25)이 기본 가장 높다. 순수 기술적 진입을 원하면 펀더·밸류를 낮추고 RS·VCP·MACD를 올린다.

## 3. 두 가지 경로

- **앱(인터랙티브):** 사이드바 토글 → 즉시 재계산. 탐색·튜닝에 최적. 바탕화면 "폭락주 스크리너" 아이콘으로 실행(최신 클라우드 스냅샷 로드).
- **워치리스트 시드(`scripts/to_watchlist.py`):** 같은 지표 엔진으로 상위 N을 골라 워치리스트로 전송.
  - 예: `python scripts/to_watchlist.py --indicators relative_strength fundamental altman_z piotroski --top 10`
  - `--list-indicators`로 지표 키 확인, `--weights key=w,...`로 가중 오버라이드, `--dry-run`으로 미리보기.
